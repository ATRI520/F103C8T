# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

STM32F103C8T6 (Blue Pill) bare-metal firmware project using STM32 HAL library. Code is generated by STM32CubeMX and built with `arm-none-eabi-gcc` via Makefile. The dev environment is VS Code with PlatformIO toolchain on Windows.

## Build Commands

```bash
# Build firmware (produces build/F103C8T.elf, .hex, .bin)
make

# Clean build artifacts
make clean

# Full rebuild
make clean && make
```

Toolchain: `arm-none-eabi-gcc` (via PlatformIO at `C:/Users/Administrator/.platformio/packages/toolchain-gccarmnoneeabi`). Linker script: `STM32F103XX_FLASH.ld` (64K FLASH, 20K RAM). Uses `-specs=nano.specs` (newlib-nano).

## Critical Rule: CubeMX Code Regions

When editing any CubeMX-generated file (`main.c`, `gpio.c`, `i2c.c`, `usart.c`, `stm32f1xx_it.c`, `stm32f1xx_hal_msp.c`), **all custom code MUST go between `/* USER CODE BEGIN ... */` and `/* USER CODE END ... */` comment pairs**. Code outside these regions will be overwritten when regenerating from the `.ioc` file. Never break this structure.

## Language

Pure C (C99). No C++ features.

## Hardware Configuration (from F103C8T.ioc)

- **USART1:** PA9 (TX), PA10 (RX), 115200 baud, handle `huart1` -- for serial debug output
- **I2C1:** PB6 (SCL), PB7 (SDA), 100kHz, handle `hi2c1` -- sensor bus
- **GPIO:** PC13 output (onboard LED, active low)
- **Clock:** HSI 8MHz (no PLL), SysTick at 8MHz
- **Debug:** SWD disabled (SYS set to No_Debug)

## Architecture

```
Core/Src/main.c          -- entry point, main loop, user business logic
Core/Src/i2c.c           -- CubeMX-generated I2C1 init (hi2c1)
Core/Src/usart.c         -- CubeMX-generated USART1 init (huart1)
Core/Src/gpio.c          -- CubeMX-generated GPIO init (PC13)
Core/Src/stm32f1xx_it.c  -- interrupt handlers
Core/Src/stm32f1xx_hal_msp.c -- HAL MSP (low-level peripheral init callbacks)
Core/Src/system_stm32f1xx.c  -- system clock setup (CMSIS)
Core/Src/syscalls.c      -- newlib syscall stubs (relevant for printf retargeting)
Core/Src/sysmem.c        -- _sbrk heap implementation
```

Headers in `Core/Inc/`, HAL driver sources in `Drivers/STM32F1xx_HAL_Driver/`, CMSIS in `Drivers/CMSIS/`.

## Adding New Source Files

When creating new `.c` files under `Core/Src/`, you **must** also add them to the `C_SOURCES` list in the `Makefile`, otherwise they won't compile. The Makefile flattens all object files into `build/`, so source filenames must be unique across all directories.

## Current Task

Read AHT10 (temperature/humidity) and SGP30 (eCO2/TVOC) sensors via I2C1 and print data over USART1 every 2 seconds. This requires:

1. **printf retargeting** -- override `_write` or `fputc` in a USER CODE region of `main.c` to route `printf` through `huart1`
2. **Sensor drivers** -- create `aht10.c/.h` and `sgp30.c/.h` under `Core/Src/` and `Core/Inc/` (hand-written HAL I2C drivers, not Arduino libraries)
3. **Main loop integration** -- init sensors in `USER CODE BEGIN 2`, read + print in the `while(1)` loop with `HAL_Delay(2000)`
